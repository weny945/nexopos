## 🔧 单位组配置方案

### 按商品类型推荐的单位组

| 商品类型 | 推荐单位组 | 基础单位 | 常用换算单位 | 说明 |
|---------|-----------|---------|-------------|------|
| **薯片、饼干** | 计数单位组-24装 | 件（袋/包） | 箱(24件) | 零食类通用 24 件/箱 |
| **方便面** | 计数单位组-混合 | 件（袋/桶） | 箱-袋装(30件)、箱-杯装(12件) | 袋装和杯装装箱数不同 |
| **面包** | 计数单位组-小批量 | 件（个） | 打(12件) | 烘焙品保质期短，小批量 |
| **奶茶（冲泡）** | 计数单位组-杯装 | 件（杯/袋） | 箱(20件)、盒(10件) | 常见 20 杯/箱 |
| **辣条** | 计数单位组-50装 | 件（包） | 箱(50件)、袋(10件) | 小包装，大箱装 |
| **花椒粉** | 重量单位组 | 克 | 千克、瓶(50g)、袋(100g) | 调味料按重量 |
| **酱油** | 容量单位组 | 毫升 | 升、瓶(500ml)、瓶(1L) | 液体调味品 |
| **可乐、雪碧** | 计数单位组-24装 | 件（瓶/听） | 箱(24件) | 标准饮料包装 |
| **啤酒** | 计数单位组-混合 | 件（瓶/听） | 箱-听装(24件)、箱-瓶装(12件) | 不同规格装箱数不同 |
| **香烟** | 计数单位组-烟草 | 件（包） | 条(10包)、箱(50条) | 烟草专用换算 |
| **水杯** | 计数单位组-通用 | 件（个） | 箱(12件)、打(12件) | 日用品通用 |

### 推荐创建的单位组列表

#### 1. 计数单位组-24装（最常用）

```
单位组名称：计数单位组-24装
描述：适用于 24 件/箱的商品（饮料、零食）

单位配置：
├─ 件（base_unit=true, value=1）
└─ 箱(24件装)（value=24）

适用商品：
├─ 可口可乐 330ML
├─ 雪碧 500ML
├─ 薯片（乐事）
├─ 饼干（奥利奥）
└─ 大部分瓶装/袋装零食
```

#### 2. 计数单位组-12装

```
单位组名称：计数单位组-12装
描述：适用于 12 件/箱的商品（啤酒、面包）

单位配置：
├─ 件（base_unit=true, value=1）
├─ 打(12件)（value=12）
└─ 箱(12件装)（value=12）

适用商品：
├─ 青岛啤酒 500ML 瓶装
├─ 雪花啤酒 500ML
├─ 面包（听装）
└─ 部分糕点
```

#### 3. 计数单位组-烟草专用

```
单位组名称：计数单位组-烟草
描述：烟草制品专用（10 包/条，50 条/箱）

单位配置：
├─ 包（base_unit=true, value=1）
├─ 条(10包)（value=10）
└─ 箱(50条)（value=500）

适用商品：
├─ 中华香烟
├─ 芙蓉王
├─ 玉溪
└─ 所有卷烟
```

#### 4. 重量单位组

```
单位组名称：重量单位组
描述：按重量销售的商品

单位配置：
├─ 克（base_unit=true, value=1）
├─ 千克（value=1000）
├─ 斤（value=500）
└─ 吨（value=1000000）

适用商品：
├─ 花椒粉
├─ 辣椒粉
├─ 散装调料
├─ 大米
└─ 面粉
```

#### 5. 容量单位组

```
单位组名称：容量单位组
描述：按容量销售的液体商品

单位配置：
├─ 毫升（base_unit=true, value=1）
├─ 升（value=1000）
└─ 加仑（value=3785）

适用商品：
├─ 酱油（散装）
├─ 食用油（散装）
├─ 醋
└─ 料酒
```

#### 6. 计数单位组-方便面专用

```
单位组名称：计数单位组-方便面
描述：方便面专用（袋装 30 件/箱，杯装 12 件/箱）

单位配置：
├─ 件（base_unit=true, value=1）
├─ 箱-袋装(30件)（value=30）
└─ 箱-杯装(12件)（value=12）

适用商品：
├─ 康师傅袋装方便面
├─ 统一袋装方便面
├─ 康师傅杯面
└─ 统一老坛杯面
```

## 💡 最佳实践建议

### 1. 分类命名规范

```
✅ 好的命名：
├─ 食品 → 休闲食品 → 膨化食品 → 薯片
├─ 饮料 → 酒类 → 啤酒 → 罐装啤酒
└─ 清晰、层级分明

❌ 不好的命名：
├─ 吃的 → 零食 → 好吃的 → 薯片
├─ 太随意，不专业
```

### 2. 分类层级建议

```
推荐：3 级分类
├─ 一级：大类（食品、饮料）
├─ 二级：中类（休闲食品、酒类）
└─ 三级：小类（膨化食品、啤酒）

不推荐超过 4 级：
└─ 过深不利于管理和查找
```

### 3. POS 显示优化

```
勾选"在 POS 显示"的分类：
├─ 一级分类：✓ 显示
├─ 二级分类：✓ 显示
└─ 三级分类：根据需要

目的：
└─ POS 收银时快速定位商品
```

### 4. 单位组复用原则

```
✅ 推荐：
└─ 相同装箱数的商品共用单位组
   ├─ 24 件/箱的饮料、零食 → 计数单位组-24装
   └─ 节省配置，统一管理

❌ 不推荐：
└─ 每个商品创建专用单位组
   └─ 配置繁琐，难以维护
```

### 5. 库存管理策略

**快消品（保质期短）：**
```
商品：面包、鲜奶
策略：
├─ 小批量采购
├─ 设置过期日期
├─ 启用"过期防止销售"
└─ 低库存预警值设置较低
```

**长保质期商品：**
```
商品：方便面、饼干
策略：
├─ 批量采购（降低成本）
├─ 库存可以多备
└─ 低库存预警值设置较高
```

**高价值商品：**
```
商品：高档香烟、洋酒
策略：
├─ 精确库存追踪
├─ 启用条形码管理
└─ 定期盘点
```

### 6. 条形码管理

```
✅ 推荐：
├─ 使用厂商标准条形码（EAN-13）
├─ 不同规格使用不同条形码
└─ 记录在商品主数据中

示例：
├─ 可口可乐 330ML 听装：6901234567891
├─ 可口可乐 330ML 箱装：6901234567892
└─ 便于扫码识别和销售
```
