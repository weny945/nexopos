## ⭐ 核心要点总结

### NexoPOS 条形码扫描能力

✅ **原生支持** - 无需任何额外开发或插件
✅ **即插即用** - USB 扫描枪直接可用
✅ **8 种格式** - EAN-13、EAN-8、CODE128、UPC-A 等
✅ **称重支持** - 完整的 PLU 码解析（电子秤条形码）
✅ **自动识别** - 200ms 防抖，自动搜索并添加商品
✅ **高性能** - 每秒可处理多次扫描

### 系统架构特点

```
前端 Vue 3 + TypeScript
    ↓ RESTful API
服务层 (30+ Services)
    ↓ Eloquent ORM
MySQL 数据库 (30+ Tables)
```

**核心服务**:
- `ProductService` - 商品和库存管理
- `BarcodeService` - 条形码生成和验证
- `ProcurementService` - 采购入库
- `OrdersService` - 销售出库

### 库存管理核心流程

⚠️ **重要概念**：商品创建 ≠ 库存数量

```
第一步：创建商品 (quantity = 0)
    ↓
    定义商品信息、单位、价格
    ❌ 此时库存为 0，无法销售

第二步：采购入库 ← 这里才填写 quantity！
    ↓
    创建采购单 → 填写数量 → 保存
    ✅ 库存更新，可以销售

第三步：POS 销售
    ↓
    扫描条形码 → 自动搜索 → 结账
    库存自动扣减

历史记录：STOCKED → SOLD → 完整审计
```

**详细说明** → 查看 [商品创建与库存管理流程](商品创建与库存管理流程.md)

**每次变动都有审计记录** → `ns_products_histories` 表

## 🚀 快速开始

### 1. 启动系统（2 分钟）

```bash
# 终端 1：启动后端
php artisan serve

# 终端 2：启动前端
npm run dev

# 浏览器访问
http://127.0.0.1:8000
```

**详细说明** → [本地启动指南](本地启动指南.md)

### 2. 创建商品并添加库存（5 分钟）

**第一步：创建商品**
```
仪表板 → 商品 → 创建
→ 填写商品名称、SKU、条形码
→ 选择分类、单位组
→ 配置单位和价格
→ 保存（此时 quantity = 0）
```

**第二步：采购入库**
```
仪表板 → 采购 → 创建采购单
→ 选择供应商
→ 添加商品
→ 填写数量 ← 这里填写！
→ 保存（库存更新）
```

**详细说明** → [商品创建与库存管理流程](商品创建与库存管理流程.md)

### 3. 使用条形码扫描（1 分钟）

1. **连接扫描枪** - USB 直接插入
2. **登录系统** - 访问 `/sign-in`
3. **打开 POS** - 访问 `/dashboard/pos`
4. **启用自动聚焦** - 点击条形码图标 🔲
5. **开始扫描** - 对准商品条形码，按触发键

**就这么简单！**

## ❓ 常见问题 FAQ

### Q1: 创建商品时在哪里填写库存数量？⭐ 最常见

**A**: **不在创建商品时填写！** 这是 NexoPOS 的设计理念。

**正确流程：**
1. 创建商品（quantity = 0）
2. 通过**采购单**添加库存 ← 在这里填写数量！

**详细说明** → [商品创建与库存管理流程](商品创建与库存管理流程.md)

### Q2: 为什么创建商品后库存为 0？

**A**: 这符合真实业务流程：
- 先定义商品信息
- 再通过采购单入库
- 然后才能销售

**如何添加库存：**
- 方式 1：仪表板 → 采购 → 创建采购单（推荐）
- 方式 2：仪表板 → 商品 → 库存调整（快速）

### Q3: Product、Category、Unit、UnitGroup 是什么关系？

**A**: 层级关系如下：
```
Category (分类)
  → Product (商品)
    → UnitGroup (单位组)
      → Units (单位)
        → ProductUnitQuantity (库存)
```

**示例：**
- 分类：饮料 → 碳酸饮料
- 商品：可口可乐
- 单位组：容量单位组
- 单位：毫升、升
- 库存：330ML(1000听)、1.25L(300瓶)

**详细说明** → [商品创建与库存管理流程](商品创建与库存管理流程.md#实体关系详解)

### Q4: NexoPOS 支持扫描枪吗？

**A**: ✅ 完全支持！无需任何插件或额外开发，USB 扫描枪即插即用。

### Q5: 支持哪些条形码格式？

**A**: 支持 8 种国际标准格式：EAN-13、EAN-8、CODE128、CODE39、CODE11、UPC-A、UPC-E、CODABAR。

### Q6: 可以使用电子秤条形码吗？

**A**: ✅ 可以！系统原生支持 PLU 码解析，自动识别重量和价格。需在设置中启用"称重条形码"功能。

### Q7: 扫描后没反应怎么办？

**A**: 检查以下几点：
1. POS 页面自动聚焦是否启用（条形码图标为蓝色）
2. 扫描枪是否配置了 Enter 后缀
3. 商品条形码是否存在于数据库
4. 商品状态是否为"可用"

### Q8: 如何生成商品条形码？

**A**: 两种方式：
1. 创建商品时手动输入条形码
2. 留空条形码字段，系统自动生成 EAN-8 码

### Q9: 可以打印条形码标签吗？

**A**: ✅ 可以！使用"打印标签"功能（`仪表板 → 商品 → 打印标签`）。

### Q10: 支持多单位管理吗？

**A**: ✅ 完全支持！同一商品可设置多个销售单位（如件/箱/吨），每个单位有独立的库存和价格。

### Q11: 库存变动有记录吗？

**A**: ✅ 有完整审计日志！每次库存变动都记录在 `ns_products_histories` 表，包含操作人、时间、变动原因、变动前后数量等。

### Q12: 采购单和库存调整有什么区别？

**A**:
- **采购单**：正式采购，记录成本、供应商，可打印单据
- **库存调整**：快速调整，用于盘点、损耗、修正

**推荐**：正式采购用采购单，临时调整用库存调整。

**详细对比** → [商品创建与库存管理流程](商品创建与库存管理流程.md#方式-1采购入库procurement-主要方式)

### Q13: 香烟、饼干、方便面等商品应该如何分类？

**A**: 推荐使用三级分类体系：

**示例分类：**
- 香烟：烟草制品 → 卷烟 → 中档香烟
- 饼干：食品 → 休闲食品 → 饼干糕点
- 方便面：食品 → 方便速食 → 方便面
- 啤酒：饮料 → 酒类 → 啤酒
- 薯片：食品 → 休闲食品 → 膨化食品

**完整分类体系** → [商品分类与单位组配置指南](商品分类与单位组配置指南.md)

### Q14: 不同商品应该选择什么单位组？

**A**: 根据商品包装特点选择：

| 商品 | 推荐单位组 | 说明 |
|------|-----------|------|
| 饮料、零食 | 计数单位组-24装 | 24件/箱 |
| 啤酒、面包 | 计数单位组-12装 | 12件/箱 |
| 香烟 | 计数单位组-烟草 | 10包/条，50条/箱 |
| 调味粉 | 重量单位组 | 按克、千克 |
| 酱油、醋 | 容量单位组 | 按毫升、升 |

**详细配置方案** → [商品分类与单位组配置指南](商品分类与单位组配置指南.md#单位组配置方案)

### Q15: 商品在库存页面能看到，但POS搜索不到？⭐ 常见问题

**A**: 这通常是商品配置问题，最常见的3个原因：

1. **Searchable 字段设置为 False（80%的情况）**
   - 进入 Dashboard → Products → 编辑商品
   - 检查 **Searchable（可搜索）** 字段，必须设置为 **Yes/True**

2. **Status 字段设置为 Unavailable（15%的情况）**
   - 检查 **Status（状态）** 字段，必须设置为 **Available（可用）**

3. **销售价格未设置或为0（5%的情况）**
   - 进入商品编辑 → Units 标签页
   - 确保每个单位的 **Sale Price（销售价格）** > 0

**快速修复步骤**：
```
1. 编辑商品
2. Searchable = Yes
3. Status = Available
4. Sale Price > 0（所有单位）
5. 保存
6. php artisan cache:clear
7. 刷新POS页面
```

**详细排查指南** → [POS商品搜索问题排查指南](POS商品搜索问题排查指南.md)
